Como Usar o Dashboard de Casos Ativos na Sua Base do Airtable

Documento 2 · Guia Completo para Novos Usuarios · Gerado em 07/05/2025

O Que e Este Dashboard?

Este dashboard e uma ferramenta web gratuita que se conecta a sua base do Airtable e gera automaticamente graficos mostrando quantos casos estiveram ativos simultaneamente em cada data. Ela calcula isso com base em duas colunas de data: uma de inicio e uma de encerramento de cada registro.

Voce pode usar para acompanhar consultorias, pacientes, projetos, atendimentos ou qualquer tipo de registro que tenha uma data de inicio e uma data de fim.

Link do Dashboard: https://rcfbeltrame.github.io/registro-consultorias-dashboard/
✓ Nao e necessario instalar nada. Basta abrir o link no navegador. Funciona no Chrome, Firefox, Edge e Safari.

O Que Voce Vai Precisar

  • Uma conta no Airtable com uma base contendo registros com duas colunas de data (inicio e fim)
  • Um Token de Acesso Pessoal do Airtable (instruções abaixo)
  • Os IDs da sua base e tabela no Airtable (instrucoes abaixo)
  • Um navegador moderno (Chrome, Firefox, Edge ou Safari)

Passo 1 — Criar um Token de Acesso no Airtable

O token e uma senha especial que permite ao dashboard ler os dados da sua base. Ela e armazenada somente no seu navegador e nunca e enviada para nenhum outro servidor.

1
Acesse o Airtable e va para as configuracoes de token Abra o navegador e acesse: https://airtable.com/create/tokens/new
Voce precisara estar logado na sua conta Airtable.
2
Nomeie o token No campo "Name your token", escreva um nome como dashboard-casos-ativos (qualquer nome que voce reconheca).
3
Configure as permissoes (Scopes) Em "Add a scope", adicione a permissao: data.records:read
Esta e a unica permissao necessaria — apenas leitura dos registros.
4
Selecione a base de dados Em "Add a base", selecione a base do Airtable que voce quer visualizar no dashboard. Voce pode adicionar mais de uma base se precisar.
5
Crie e copie o token Clique em "Create token". O token sera exibido uma unica vez — ele tem o formato patXXXXXXXX.XXXXX...
Copie e salve em lugar seguro. Ele nao sera exibido novamente.
⚠ Nao compartilhe seu token com outras pessoas. Quem tiver o token pode ler os dados da sua base.

Passo 2 — Encontrar os IDs da Sua Base e Tabela

O Airtable identifica cada base e tabela com um codigo unico. Voce precisa desses codigos para configurar o dashboard.

1
Abra sua base no Airtable e olhe a URL do navegador A URL tem o seguinte formato:
https://airtable.com/appo7XXXXXXXX/tblYYYYYYYYY/viwZZZZZZZZZ
2
Identifique cada parte da URL
Parte da URLComeca comNomeExemplo
ID da BaseappBase IDappio7ZC2rZahAJWl
ID da TabelatblTable IDtblYq7mKJNQRaiHA3
ID da ViewviwView IDviwxn35nLyYeQpk89
3
Anote os nomes das duas colunas de data Voce precisara do nome exato (com acentos, maiusculas e minusculas) das colunas que representam:
• Data de inicio do registro (ex: "Data de Entrada", "Inicio", "Data Avaliacao")
• Data de encerramento do registro (ex: "Data de Saida", "Fim", "Data Alta") — registros sem data de fim serao tratados como ainda ativos.
💡 Dica: O nome da view e opcional. Se voce quiser usar todos os registros da tabela, basta deixar em branco ou usar o nome da view principal. Se quiser filtrar por uma view especifica do Airtable, use o ID da view (comeca com viw).

Passo 3 — Configurar o Dashboard

1
Abra o dashboard no navegador Acesse: https://rcfbeltrame.github.io/registro-consultorias-dashboard/
Na primeira vez, voce vera um formulario de configuracao.
2
Preencha os campos do formulario
Campo do formularioO que colocar
Token de AcessoO token que voce copiou no Passo 1 (comeca com pat...)
ID da BaseO codigo que comeca com app... (da URL do Airtable)
ID da TabelaO codigo que comeca com tbl... (da URL do Airtable)
ID/Nome da ViewO codigo que comeca com viw... ou o nome da view (opcional)
Campo de InicioNome exato da coluna de data de inicio (ex: "Data Avaliacao")
Campo de FimNome exato da coluna de data de encerramento (ex: "Data Alta")
3
Clique em "Carregar Dados" O dashboard ira buscar todos os registros da sua base e gerar os graficos automaticamente. Isso pode levar alguns segundos dependendo da quantidade de registros.
✓ Suas configuracoes sao salvas automaticamente no seu navegador. Na proxima vez que abrir o dashboard, os dados ja estarao pre-preenchidos — basta clicar em "Carregar Dados".

Passo 4 — Usando o Dashboard

Navegando pelos periodos

Na parte superior do dashboard, ha abas com os anos detectados nos seus dados (ex: 2023, 2024, 2025) e uma aba chamada "Periodo Total" que mostra todos os anos juntos. Clique em qualquer aba para ver apenas aquele periodo.

Tipos de visualizacao

Para cada periodo, voce pode escolher como visualizar os dados:

VisualizacaoO que mostra
DiarioQuantos casos estavam ativos em cada dia. No "Periodo Total", a curva e suavizada com media de 14 dias para facilitar a leitura.
SemanalO pico de casos ativos em cada semana
Mensal (Max)O pico maximo de casos em cada mes
Mensal (Media)A media diaria de casos em cada mes
Mensal (Mediana)A mediana diaria de casos em cada mes

Cards de estatisticas

Acima do grafico, ha cartoes com as principais metricas do periodo selecionado: pico de casos, media diaria, mediana diaria, total de registros e casos ativos hoje.

Baixando os graficos

Cada grafico tem um botao de download (icone de seta para baixo). Clique nele para salvar o grafico como imagem PNG com fundo branco, pronta para usar em apresentacoes ou relatorios.

Solucao de Problemas Comuns

ProblemaO que fazer
"Nenhum registro valido. Verifique os IDs dos campos."Os nomes das colunas de data estao errados. Verifique no Airtable o nome exato (com acentos e maiusculas) e corrija no formulario.
"Erro ao carregar dados" ou pagina em brancoVerifique se o token esta correto e se a base selecionada na criacao do token e a mesma do ID informado.
Os dados nao atualizamClique no botao "Recarregar" na barra superior para buscar os dados mais recentes do Airtable.
Quero alterar as configuracoesClique em "Alterar Configuracoes" na barra superior para editar token, base ou campos.
Um colega quer usar a mesma ferramentaBasta enviar o link. Na primeira abertura, ele configura com o token e base dele. As configuracoes de cada pessoa ficam salvas separadamente no navegador de cada um.

Privacidade e Seguranca

  • O token e as configuracoes ficam armazenados apenas no seu navegador (localStorage). Nenhum dado e enviado para servidores externos.
  • O codigo do dashboard e publico no GitHub, mas os dados da sua base nunca sao acessados por terceiros — a conexao e feita diretamente do seu navegador para o Airtable.
  • Voce pode revogar o token a qualquer momento nas configuracoes do Airtable, sem afetar outros usuarios.
  • Cada usuario precisa criar e usar o proprio token.
