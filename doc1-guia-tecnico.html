1. Visao Geral do Projeto

Dashboard web estatico hospedado no GitHub Pages que conecta ao Airtable via API REST para calcular e visualizar a densidade de casos clinicos ativos simultaneamente por dia.

  • URL publica: https://rcfbeltrame.github.io/registro-consultorias-dashboard/
  • Repositorio GitHub: github.com/rcfbeltrame/registro-consultorias-dashboard
  • Arquivo principal: index.html (arquivo unico autocontido)
  • Branch de deploy: main (GitHub Pages configurado na raiz)

3. Fluxo de Dados

  1. Usuario insere token e configuracoes na tela inicial do app
  2. App salva no localStorage com prefixo dash_ (ver secao 4)
  3. App chama API do Airtable com paginacao por offset ate buscar todos os registros
  4. Para cada registro: calcula intervalo entre campo-inicio e campo-fim (se fim ausente = caso ainda ativo, usa data atual)
  5. Monta mapa {"YYYY-MM-DD": count} — para cada dia, quantos casos estavam ativos
  6. Agrega por semana/mes, calcula media e mediana diaria; renderiza graficos de barras via Chart.js

5. Logica JavaScript — Pontos Criticos

5.1 API retorna campo por NOME, nao por ID

// CORRETO — usar o nome exato do campo como chave:
const startVal = record.fields[startField]; // ex: record.fields["Data Avaliacao"]
const endVal   = record.fields[endField];   // ex: record.fields["Data Alta Cardiologica"]
// ERRADO — a API NAO retorna campos por ID (fldXXX...)

5.2 Paginacao completa da API

async function fetchAll(token, base, table, view) {
  let records = [], offset = null;
  do {
    const url = "https://api.airtable.com/v0/" + base + "/" + table
              + "?view=" + encodeURIComponent(view)
              + (offset ? "&offset=" + offset : "");
    const res  = await fetch(url, { headers: { Authorization: "Bearer " + token } });
    const data = await res.json();
    records = records.concat(data.records || []);
    offset  = data.offset;
  } while (offset);
  return records;
}

5.3 Calculo de casos ativos por dia

const dm = {}; // mapa data -> count
records.forEach(r => {
  const start = new Date(r.fields[startField]);
  const end   = r.fields[endField] ? new Date(r.fields[endField]) : new Date(); // hoje se vazio
  for (let d = new Date(start); d <= end; d.setDate(d.getDate() + 1)) {
    const key = d.toISOString().slice(0, 10); // "YYYY-MM-DD"
    dm[key] = (dm[key] || 0) + 1;
  }
});

5.4 Suavizacao (Rolling Average 14 dias) — APENAS Periodo Total

Regra: suavizacao aplicada SOMENTE quando CUR_YEAR === "total" AND CUR_VIEW === "diario". Anos individuais usam dados brutos para preservar dias de baixa demanda que seriam diluidos pela media.

function rollingAvg(keys, dm, windowSize) {
  return keys.map((k, i) => {
    const slice = keys.slice(Math.max(0, i - windowSize + 1), i + 1);
    const sum   = slice.reduce((a, b) => a + (dm[b] || 0), 0);
    return Math.round((sum / slice.length) * 10) / 10;
  });
}
// Condicional de aplicacao:
if (CUR_YEAR === "total" && CUR_VIEW === "diario") {
  vals  = rollingAvg(keys, dm, 14);
  label = "Media Movel 14 Dias";
} else {
  vals  = keys.map(k => dm[k] || 0);
  label = "Casos Ativos por Dia";
}

5.7 Download de grafico como PNG com fundo branco

function downloadChart() {
  const canvas = document.getElementById("myChart");
  const tmp    = document.createElement("canvas");
  tmp.width    = canvas.width;
  tmp.height   = canvas.height;
  const ctx    = tmp.getContext("2d");
  ctx.fillStyle = "#ffffff";
  ctx.fillRect(0, 0, tmp.width, tmp.height);
  ctx.drawImage(canvas, 0, 0);
  const a      = document.createElement("a");
  a.download   = "grafico_" + CUR_YEAR + "_" + CUR_VIEW + ".png";
  a.href       = tmp.toDataURL("image/png");
  a.click();
}

7. Funcionalidades do Dashboard

  • Tabs de periodo: Um tab por ano detectado nos dados + "Periodo Total" — gerados automaticamente
  • Tabs de visualizacao: Diario, Semanal, Mensal (Maximo), Mensal (Media), Mensal (Mediana)
  • Cards de estatisticas: Pico de casos, Media diaria, Mediana diaria, Total de registros, Casos ativos hoje
  • Graficos: Sempre barras (Chart.js), nunca linhas
  • Suavizacao: Media movel 14 dias apenas em Periodo Total + Diario
  • Download: Botao exporta PNG com fundo branco para cada grafico
  • Recarregar dados: Botao na topbar recarrega da API sem apagar configuracoes

8. Passo a Passo para Recriar do Zero

1
Criar repositorio GitHub publico. Settings → Pages → Source: Deploy from branch → Branch: main → /(root) → Save.
2
Criar index.html no repositorio com o codigo completo autocontido (sem arquivos CSS/JS separados — tudo em um unico arquivo).
3
Editar via GitHub Web: clicar no icone de lapis → editor CodeMirror 6 abre. Para substituir tudo via automacao: focar em .cm-content, pressionar Cmd+A, depois executar document.execCommand("insertText", false, novoHTML).
4
Commitar: clicar "Commit changes..." → digitar mensagem no modal → clicar "Commit changes" (botao verde). GitHub Pages atualiza em ~30-60 segundos.
5
Verificar deploy: abrir URL do Pages, abrir DevTools (F12) → Console. Se "Failed to fetch", aguardar deploy e recarregar. Se "Nenhum registro valido", verificar se os nomes dos campos estao corretos.
